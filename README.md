# FrameNote

**FrameNote**는 Google Meet, Zoom 같은 화상회의와 일반 화면 작업을 저장하는 macOS/Windows 데스크톱 녹화 앱입니다. 화면 또는 창, 시스템 오디오, 마이크를 한 번에 녹화하고, 장시간 녹화에서도 메모리가 계속 늘지 않도록 녹화 데이터를 즉시 디스크에 기록합니다.

현재 버전은 **0.3.0**입니다. Electron, React, TypeScript로 사용자 화면을 구성하고, 운영체제별 네이티브 모듈을 함께 사용해 시스템 오디오와 macOS 창 커서 정책을 처리합니다.

## 이 앱이 해결하는 문제

화상회의를 녹화할 때는 발표 화면뿐 아니라 상대방의 음성과 내 마이크를 함께 담아야 합니다. FrameNote는 화면 캡처, 시스템 오디오, 마이크, 파일 저장을 하나의 흐름으로 묶고 다음 원칙을 따릅니다.

- 회의 녹화에 필요한 입력을 한 화면에서 선택하고 상태를 확인합니다.
- 녹화 전체를 메모리에 보관하지 않고 작은 청크 단위로 디스크에 저장합니다.
- 녹화 종료나 앱 종료 중에도 가능한 한 파일을 안전하게 마무리합니다.
- MP4를 우선하되 현재 환경에서 지원하지 않으면 WebM으로 대체합니다.
- macOS와 Windows의 권한 및 오디오 캡처 차이를 앱 내부에서 처리합니다.

## 주요 기능

- 전체 화면 또는 앱 창 선택과 실시간 미리보기
- 회의 모드: 시스템 오디오 + 소음 억제 마이크가 기본값
- 화면 모드: 화면 + 시스템 오디오가 기본값
- 파일 형식과 코덱을 각각 선택: MP4 + H.264/AAC 또는 WebM + VP9/Opus·VP8/Opus
- 자동 형식: 범용 MP4를 먼저 사용하고, 미지원 환경에서는 효율적인 WebM으로 대체
- 효율(720p), 균형(1080p), 고화질(1440p), 부드럽게(1080p 60fps), 4K(2160p) 프리셋
- 절약·균형·최상 용량 전략과 시간당 예상 파일 크기
- 0·3·5초 시작 카운트다운
- 녹화 일시정지/재개, 저장 폴더 선택, 완료 파일 바로 보기
- macOS 화면/마이크 권한 상태, 설정 바로가기, 오래된 화면 권한 연결 초기화
- macOS Intel/Apple Silicon 및 Windows x64/ARM64 패키징

## 빠른 시작

Node.js 24 이상을 권장합니다.

```bash
npm install
npm run dev
```

프로덕션 코드와 테스트를 확인하려면:

```bash
npm run build
```

## 기술 구성

- Electron 43 + electron-vite
- React 19 + TypeScript 5
- Chromium MediaRecorder 기반 화면·마이크 인코딩
- macOS ScreenCaptureKit + Core Audio 프로세스 탭 기반 시스템 오디오 헬퍼
- Windows WASAPI 기반 시스템 오디오 헬퍼
- Vitest 기반 단위·회귀 테스트

## 프로젝트 구조

```text
src/main/       Electron 메인 프로세스, 권한·파일·네이티브 프로세스 관리
src/preload/    renderer에 노출하는 안전한 IPC 경계
src/renderer/   React UI와 녹화 제어
src/shared/     프로세스 간 계약, 설정, 상태 모델
native/         macOS·Windows 네이티브 캡처 코드
scripts/        네이티브 빌드와 설치 파일 패키징
tests/          녹화·권한·설정·패키징 회귀 테스트
docs/           제품 사양과 아키텍처 문서
```

설치 파일은 각 운영체제에서 빌드하는 것이 가장 안정적입니다.

```bash
# macOS: 범용 DMG + ZIP (Intel + Apple Silicon)
npm run package:mac

# Developer ID가 없는 로컬 배포: 전체 ad-hoc 서명 범용 DMG
npm run package:mac:adhoc

# Windows: NSIS 설치 프로그램 (x64, ARM64)
npm run package:win
```

태그를 푸시하거나 GitHub Actions의 `Build desktop installers` 워크플로를 수동 실행하면 macOS와 Windows 러너가 각각 설치 파일을 만듭니다. macOS 릴리스는 권한 신원을 버전 간 유지하기 위해 Developer ID 서명과 공증이 필수이며, 다음 GitHub Actions secret이 없으면 빌드를 의도적으로 중단합니다.

- `MAC_CSC_LINK`, `MAC_CSC_KEY_PASSWORD`: Developer ID Application 인증서와 암호
- `APPLE_ID`, `APPLE_APP_SPECIFIC_PASSWORD`, `APPLE_TEAM_ID`: Apple 공증 계정

`package:mac:adhoc`은 인증서가 없는 로컬 테스트 전용입니다. Windows 공개 배포에는 별도의 코드 서명 인증서를 구성해야 합니다.

## 첫 실행 권한

### macOS

#### Gatekeeper 경고 안내

`package:mac:adhoc`으로 만든 macOS 설치 파일은 Apple Developer ID 인증서로 서명하거나 Apple 공증을 받은 공개 배포본이 아닙니다. 이 파일을 처음 실행하면 macOS에서 개발자를 확인할 수 없거나 Apple이 악성 소프트웨어 여부를 확인할 수 없다는 Gatekeeper 경고가 표시될 수 있습니다.

이 유형의 경고는 FrameNote에서 악성코드가 검출됐다는 뜻이 아니라, Apple이 배포자의 Developer ID와 공증 티켓을 확인할 수 없다는 뜻입니다. 따라서 앱 기능상의 오류는 아니지만, 경고를 무시하는 행동 자체가 파일의 안전을 보장하지는 않습니다. 반드시 이 저장소의 공식 **GitHub Releases**에서 받은 FrameNote 설치 파일이고 다운로드 후 변경되지 않은 파일이라고 확신할 때만 실행하세요.

반대로 macOS가 앱이 **손상됨**, **컴퓨터를 손상시킴** 또는 **악성 소프트웨어가 감지됨**이라고 경고하면 아래 절차로 우회하지 마세요. 파일을 삭제하고 이 저장소에 문제를 신고해야 합니다. 이런 경고는 파일의 변조·손상, 악성 콘텐츠 감지 또는 인증 취소를 뜻할 수 있습니다.

공식 Releases에서 받은 파일인데 실행이 차단됐다면:

1. FrameNote 실행을 한 번 시도해 Gatekeeper 경고를 확인합니다.
2. 시스템 설정 → 개인정보 보호 및 보안으로 이동합니다.
3. 보안 영역에서 FrameNote의 `그래도 열기`를 선택한 다음, 다시 나타나는 경고에서 `열기`를 선택합니다.

메뉴 이름과 절차는 macOS 버전에 따라 조금 다를 수 있습니다. 자세한 내용은 Apple의 [Mac에서 앱을 안전하게 열기](https://support.apple.com/ko-kr/102445) 안내를 참고하세요. Developer ID 서명과 Apple 공증을 적용한 정식 배포본에서는 Gatekeeper가 개발자 신원과 변조 여부를 확인할 수 있습니다.

1. DMG에서 FrameNote를 실행하고 `응용 프로그램으로 이동`을 누릅니다. 앱이 설치된 위치에서 자동으로 다시 실행됩니다.
2. 앱에서 `화면 접근 허용`을 누른 다음 시스템 설정 → 개인정보 보호 및 보안 → 화면 및 시스템 오디오 기록에서 FrameNote를 허용합니다.
3. 마이크 녹음을 사용한다면 마이크 권한도 허용합니다.
4. 권한을 바꾼 후 앱을 완전히 종료하고 다시 실행합니다.

권한을 켰는데도 화면 기록 알림이 반복되면 앱 설정의 `macOS 권한 → 권한 연결 초기화`를 사용합니다. 확인 후 현재 번들 ID와 이전 버전의 화면 기록 항목을 초기화하고 앱을 재실행한 다음, macOS 설정을 자동으로 엽니다. 이 작업은 녹화 파일이나 일반 앱 설정을 지우지 않습니다.

macOS 14.2 이상에서는 첫 시스템 오디오 녹화 때 별도의 오디오 캡처 권한 창이 나타납니다. 앱 설정의 `macOS 권한 → 시스템 오디오`에서 해당 설정 화면을 다시 열 수 있습니다.

macOS 13 이상을 주 지원합니다. 12.7.6 이하는 Apple API 제약 때문에 BlackHole 같은 가상 오디오 장치 없이 시스템 오디오를 가져올 수 없습니다. macOS 14.2 이상용 `NSAudioCaptureUsageDescription`도 패키지에 포함되어 있습니다.

### Windows

Windows 10 이상을 지원합니다. 시스템 오디오에는 Electron/Chromium의 loopback 캡처를 사용합니다. Windows 설정에서 데스크톱 앱의 마이크 접근이 꺼져 있으면 켜야 합니다.

## 성능과 메모리 설계

- 캡처와 MediaRecorder 인코딩은 Chromium의 미디어 스레드와 가능한 경우 하드웨어 가속을 사용합니다.
- React UI와 비동기 파일 기록은 renderer/main 프로세스로 분리했습니다.
- 녹화는 1초 청크로 기록하고 전체 파일 Blob을 메모리에 보관하지 않습니다.
- 녹화 중에는 `.partial` 파일로 기록하고 정상 종료·동기화가 끝난 뒤 최종 확장자로 원자적으로 전환합니다.
- 파일 기록 대기량이 32MB를 넘으면 내용을 조용히 누락하지 않도록 녹화를 종료하고 현재까지의 데이터를 안전하게 마무리한 뒤 사용자에게 알립니다.
- 창 포커스에서는 권한 상태만 가볍게 확인하며, 화면 썸네일은 권한이 새로 허용되거나 사용자가 새로고침할 때만 다시 생성합니다.
- 화질 프리셋·코덱 효율·용량 전략을 합쳐 목표 비트레이트를 정하고, 전체 녹화 파일을 메모리에 쌓지 않습니다.

상세 구조는 [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md), 요구사항은 [docs/SPEC.md](docs/SPEC.md)를 참고하세요.

## 알려진 제약

- DRM 보호 영상, 회사 관리 정책, 일부 보안 앱은 화면 또는 오디오 캡처를 막을 수 있습니다.
- MP4 MediaRecorder 지원 여부는 운영체제의 Chromium 미디어 기능에 따라 달라집니다. 미지원 장치에서는 자동으로 WebM을 사용합니다.
- 전체 ad-hoc 서명 빌드는 같은 앱 파일을 다시 실행할 때 권한이 유지되지만, 새 버전으로 교체하면 macOS가 다른 코드 신원으로 판단해 권한을 다시 물을 수 있습니다. 공개 배포 빌드는 고정된 번들 ID와 Developer ID로 서명하고 공증해야 합니다.

## 개발 이력

아래 내용은 저장소의 버전 변경 지점과 커밋 기록을 사용자 관점에서 정리한 것입니다. 아직 Git 태그 기반의 정식 릴리스 기록은 아닙니다.

### 0.3.0 — FrameNote 리브랜딩

- 앱 이름, 번들 ID, 설치 파일명과 기본 녹화 폴더를 FrameNote로 변경했습니다.
- 캡처 프레임과 노트를 결합한 새 앱 아이콘을 적용했습니다.
- macOS·Windows 네이티브 오디오 헬퍼와 내부 식별자를 새 브랜드에 맞게 통일했습니다.
- 이전 버전의 macOS 화면 녹화 권한 항목도 초기화할 수 있도록 기존 번들 ID를 마이그레이션 목록에 유지했습니다.

### 0.2.4 — 화면 표시 안정화

- 좁은 영역에서 화질 메타데이터가 UI 밖으로 넘치던 문제를 수정했습니다.
- 해당 레이아웃이 다시 깨지지 않도록 회귀 테스트를 추가했습니다.

### 0.2.3 — 캡처 상태와 창 녹화 개선

- 실제 캡처 상태와 화면에 표시되는 상태를 동기화했습니다.
- macOS에서 특정 창 녹화 시 MinuteFrame 커서 정책을 적용하는 네이티브 모듈을 추가했습니다.
- 소스 선택·새로고침과 코덱 호환성 처리를 보강했습니다.
- 미리보기 재시도 횟수를 제한하고 네이티브 모듈 서명 검증을 강화했습니다.
- 녹화 완료 파일을 Finder 또는 Explorer에서 안정적으로 표시하도록 개선했습니다.

### 0.2.2 — 선택한 소스의 시스템 오디오 캡처

- 선택한 화면이나 창에 맞춰 회의 오디오를 캡처하는 네이티브 경로를 추가했습니다.
- macOS에는 화면용 ScreenCaptureKit과 창·앱용 Core Audio 프로세스 탭을 결합한 오디오 헬퍼를, Windows에는 WASAPI 기반 헬퍼를 도입했습니다.
- 네이티브 오디오를 브라우저 녹화 스트림으로 전달하는 AudioWorklet 경로를 추가했습니다.
- 헬퍼 프로세스 종료, 오류 전달, macOS 릴리스 패키징을 보강했습니다.

### 0.2.1 — 녹화 복구와 성능 안정화

- 녹화 시작·중지·종료 흐름의 경합 조건을 줄이고 앱 종료 시 파일 마무리를 강화했습니다.
- 게시 또는 파일 전환에 실패해도 복구 가능한 `.partial` 녹화를 보존하도록 개선했습니다.
- 설정 저장을 직렬화하고 잘못된 저장값을 안전하게 정규화했습니다.
- 권한 확인과 화면 소스 새로고침이 중복 실행되지 않도록 조정했습니다.
- 디스크 기록의 백프레셔와 실패 처리를 보강했습니다.

### 0.2.0 — MinuteFrame 기능 확장

- 720p부터 4K까지의 화질 프리셋과 용량 전략, 예상 파일 크기를 추가했습니다.
- 회의/화면 모드, 녹화 시작 카운트다운, 권한 복구 기능을 추가했습니다.
- macOS 권한과 서명·공증을 고려한 배포 흐름을 강화했습니다.

### 0.1.0 — 최초 구현

- 전체 화면·창 선택, 실시간 미리보기, 시스템 오디오·마이크 녹화를 구현했습니다.
- 녹화 데이터를 1초 청크로 디스크에 기록하는 파일 저장 구조를 도입했습니다.
- 녹화 일시정지·재개·종료, 설정 저장, 파일 형식과 화질 선택을 구현했습니다.
- macOS와 Windows 설치 파일을 만드는 GitHub Actions 워크플로를 추가했습니다.

## 품질 확인

```bash
npm run typecheck
npm test
npm run build
```
