# FrameNote 제품 사양

## 목표

macOS와 Windows에서 Google Meet, Zoom 등 화상회의와 사용자의 화면을 안정적으로 저장하는 데스크톱 앱을 제공한다.

## 필수 요구사항

1. 전체 화면 또는 개별 창을 선택해 녹화할 수 있어야 한다.
2. 시스템 오디오와 마이크를 각각 켜고 끌 수 있어야 하며, 회의 모드의 기본값은 둘 다 켜짐이다.
3. 파일 형식과 코덱은 별도 설정으로 제공한다. 자동 형식은 재생 호환성을 우선해 MP4(H.264/AAC)를 사용하고, 현재 OS가 이를 지원하지 않으면 WebM(VP9/Opus), WebM(VP8/Opus) 순서로 대체한다.
4. 사용자는 파일 형식, 코덱, 720p~4K 품질 프리셋, 용량 전략을 선택할 수 있다.
5. 녹화 시작, 일시정지, 재개, 종료를 직관적으로 제어할 수 있다.
6. 녹화 데이터는 메모리에 누적하지 않고 작은 청크로 즉시 디스크에 기록한다.
7. 화면 캡처·인코딩과 파일 I/O가 UI 응답성을 방해하지 않도록 프로세스 경계를 분리한다.
8. macOS 권한 상태를 보여주고 필요한 시스템 설정으로 이동할 수 있어야 하며, 업데이트 후 오래된 화면 기록 권한 연결을 사용자가 명시적으로 초기화할 수 있어야 한다.
9. 결과 폴더를 선택하고 완료된 파일을 Finder 또는 Explorer에서 열 수 있어야 한다.
10. macOS Intel/Apple Silicon 및 Windows x64/ARM64용 패키징 구성을 제공한다.
11. 앱 시작 시 화면 캡처 권한 팝업을 자동으로 띄우지 않고, 사용자 동작으로만 요청한다.
12. 사용자는 녹화 전 0초, 3초, 5초 카운트다운을 선택할 수 있다.
13. 사용자는 시스템 오디오와 마이크가 혼합된 음성을 영상과 별도 파일로 추출하도록 선택할 수 있어야 한다. M4A(AAC)를 우선하고 현재 환경에서 지원하지 않으면 WebM(Opus)으로 대체한다.
14. 선택적인 음성 파일 생성에 실패해도 정상적으로 저장할 수 있는 영상 파일은 보존해야 한다.

## 기본값

- 용도: 회의 녹화
- 품질: 균형(1080p, 30fps, 6Mbps)
- 파일 형식: 자동(MP4 우선)
- 시스템 오디오: 켜짐
- 마이크: 켜짐
- 코덱: 자동(호환성 우선)
- 용량 전략: 균형
- 시작 카운트다운: 3초
- 오디오: 48kHz, 192kbps, echo cancellation/noise suppression/auto gain 적용
- 음성 파일 추출: 꺼짐

사용자가 특정 코덱을 직접 선택했는데 현재 장치가 지원하지 않으면 다른 코덱으로 조용히 바꾸지 않고 안내한다. 자동 모드에서만 호환 순서에 따른 대체를 허용한다.

## 지원 범위

- macOS 13 이상, Windows 10 이상을 주 지원 대상으로 한다.
- macOS 12.7.6 이하는 OS API 제약 때문에 별도의 가상 오디오 장치 없이는 시스템 오디오를 캡처할 수 없다.
- DRM 보호 영상이나 관리 정책으로 막힌 앱은 운영체제가 캡처를 차단할 수 있다.
